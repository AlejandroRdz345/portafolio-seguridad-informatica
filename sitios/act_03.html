3. Anatomía de un comando iptables

Iptables -A input -p tcp -m multiport –dports 80,43 -j ACCEPT

4. Este comando permite:

Crea una regla, permite entrada a los puertos http/https

5. Variables y opciones comunes

A) Limitar intentos

–Limit 5/minute

B) Filtrar por IP de origen

-S 192.168.1.0/24

C) Ver solo números, sin DNS (ni resolución de puertos)

-L -n

D) Ver reglas con contenedores

-L -v

6. ¿Qué hace esta regla?

Iptables -A INPUT -i eth0 -p tcp -m multiport –dports 22,80,443 \ -m state –state NEW,ESTABLISHED -j ACCEPT
Permite trafico tcp entrante por la interfaz eth0 por medio de los puertos 22(ssh), 80(http) y 443(https)

Ejercicios de traducción de políticas

7. Permitir tráfico http entrante

Iptables -A input -p tcp –dport 80 -j ACCEPT

8. Permitir todo el tráfico

Iptables -A output -j ACCEPT

9. Permitir ssh solo desde la IP 192.168.1.50

Iptables -A input -p tcp –dport 22 -S 192.168.1.50

10. Permitir tráfico TCP entrante a puertos 80 y 443 solo si es conexión establecida o relacionada

Iptables -A input -p tcp –dports 80,443 -m state –state ESTABLISHED,RELATED

11. Permitir tráfico tcp entrante por eth0 a 22, 80 y 443, registrar intentos y permitir solo NEW y ESTABLISHED

Iptables -A INPUT -p tcp -i eth0 -–dports 22,80,443 \ -m state –state NEW,ESTABLISHED -j log –log-ports "conexion nueva" .j ACCEPT
